## 1. Общие принципы
- Cursor-first: 80% кода генерируется Cursor. Никогда не пиши вручную то, что можно получить через хороший промпт.
- Один PR — одна задача. Не больше 300 строк изменений.
- Всегда пиши тесты (pytest) для новых функций и эндпоинтов.
- Типизация обязательна (Python 3.12 + from __future__ import annotations).
- Безопасность ФЗ-152 — любое действие с ПДн должно проходить через существующий security модуль.
- Русский язык — все логи, комментарии, переменные, которые видит пользователь — только на русском.

- Используй только существующий SecurityContext и PermissionDependency.

### 1. Как писать промпты (золотые правила)

Хороший промпт всегда содержит:
1. Роль — «Ты senior Python-разработчик с опытом FastAPI + Aiogram 3 + LangChain»
2. Контекст проекта — @workspace RULES.md + название модуля
3. Что именно нужно сделать (одна задача!)
4. Технические ограничения (какие библиотеки можно/нельзя использовать)
5. Примеры (если есть)
6. Формат ответа — «Верни только код + краткое объяснение»

Запрещено:
- Просить «сделай весь модуль» за один раз
- Просить «сделай красиво» без конкретных требований
- Игнорировать существующие классы и паттерны

### 2. Полезные команды в Cursor

- Cmd+K (Composer) — для больших изменений
- Cmd+L — чат с полным контекстом файла
- @rules — всегда добавляй в промпт (Cursor автоматически подхватит RULES.md)
- @file — ссылка на конкретный файл
- @folder — весь модуль
