# Техническое задание на развитие модулей SNARK Portal

Документ подготовлен на основе лучших практик **Bitrix24** (корпоративный портал, CRM, задачи, HR) и **amoCRM** (воронки, сделки, автоматизация, коммуникации). Цель — развивать SNARK как единую цифровую среду компании без перегруза интерфейса.

---

## 1. Принципы продукта (из Bitrix24 + amoCRM)

SNARK развивается как единая рабочая среда, а не набор разрозненных страниц. За основу берём подход Bitrix24 — всё важное на одном экране: лента, задачи, заявки, справочник. Из amoCRM перенимаем идею **карточки сущности** как центра работы: открыл заявку, сотрудника или задачу — и все действия, статус и история собраны в одном месте. На дашборде это выражается в быстрых действиях и контекстных карточках, без лишних переходов.

**Роли и права.** В Bitrix24 доступ строится через группы и отделы, в amoCRM — через роли менеджеров и видимость воронки. В SNARK три базовые роли — admin, hr_manager, employee — с расширением ACL по модулям: не каждый видит админку, не каждый может согласовывать отпуска, но каждый понимает, что ему доступно.

**Автоматизация.** Bitrix24 силён роботами и бизнес-процессами, amoCRM — Digital Pipeline и задачами при смене этапа сделки. Для SNARK на первом этапе достаточно очереди событий (Redis) и webhook'ов: смена статуса заявки → уведомление, создание протокола → задачи исполнителям. Сложные сценарии — позже, без усложнения интерфейса сейчас.

**Прозрачность статусов.** Пользователь не должен гадать, «в отпуске» человек или «отошёл». Статусы заявок, задач и присутствия сотрудников единообразны в списках и в карточках — как стадии воронки в amoCRM или стадии задачи в Bitrix24.

**Мобильность.** Bitrix24 опирается на мессенджер и push, amoCRM — на мобильное приложение. SNARK на текущем этапе — адаптивный веб-интерфейс; PWA и push-уведомления заложены в дорожную карту, но не блокируют запуск основных модулей.

## 2. Модуль «Портал и дашборд»

**Референс Bitrix24:** главная лента, виджеты, быстрые ссылки.  
**Референс amoCRM:** стартовый экран с KPI и задачами на сегодня.

### Функции MVP (есть / в работе)
- Приветствие, дата, дни рождения
- Быстрые действия (заявка, справочник, новости)
- Виджеты: новости, новые сотрудники, задачи

### Функции v2
- Настраиваемые виджеты админом (drag-and-drop)
- Персональная лента: заявки, согласования, упоминания
- KPI-блоки для руководителей (открытые заявки, просроченные задачи)

### API / данные
- `GET /api/dashboard` — агрегат виджетов
- Таблица `dashboard_widgets` (userId, widgetType, sortOrder, config JSON)

### Критерии приёмки
- Время загрузки дашборда < 2 с при 50+ сотрудниках
- Быстрые действия редактируются в админке без деплоя

## 3. Модуль «Сотрудники и оргструктура»

**Референс Bitrix24:** структура компании, профили, статусы присутствия.  
**Референс amoCRM:** карточка контакта с историей касаний.

### Функции MVP
- Справочник сотрудников, фильтры по отделам
- Карточка сотрудника (контакты, отдел, руководитель)
- Профиль: фото с редактором, телефон, статус (офис / удалёнка / отпуск)
- Единый формат ФИО: Фамилия Имя Отчество

### Функции v2
- Оргдиаграмма (дерево подразделений) с drill-down
- История изменений профиля (audit log)
- Интеграция с AD/LDAP для синхронизации ФИО и отделов
- Вкладка «Активность»: заявки, задачи, отпуска сотрудника

### API
- `GET /api/contacts`, `GET /api/contacts/:id`
- `PATCH /api/users/me`, `POST /api/users/me/avatar`
- `GET /api/departments/tree`

### Критерии приёмки
- Статус в списке и карточке совпадает
- Аватар сохраняется после редактирования и виден в справочнике

## 4. Модуль «Поддержка (Service Desk)»

**Референс Bitrix24:** Helpdesk, заявки, SLA.  
**Референс amoCRM:** воронка обращений, ответственный, теги.

### Функции MVP (реализовано частично)
- Создание заявки: категория, приоритет, тема, описание
- **Управление категориями** в админке (`/admin/support-categories`)
- Список «Мои заявки» / админ-очередь
- Взятие заявки в работу

### Функции v2 (по образцу Bitrix24 Helpdesk + amoCRM воронки)
- SLA по категории (время первого ответа, время решения)
- Статусы: новая → в работе → ожидание ответа → решена → закрыта
- Комментарии и вложения в заявке (лента как в карточке amoCRM)
- Назначение ответственного по категории (маршрутизация)
- Email-уведомления автору при смене статуса
- Отчёты: среднее время решения, нагрузка по категориям

### Модель данных
- `ticket_categories` (slug, label, description, isActive, sortOrder) — **есть**
- `tickets` — **есть**
- `ticket_comments`, `ticket_attachments` — v2
- `ticket_sla_policies` — v2

### API
- `GET /api/ticket-categories` — активные категории
- `POST /api/tickets`, `GET /api/tickets`
- Admin: `CRUD /api/admin/ticket-categories`, `PATCH /api/admin/tickets/:id`

### Критерии приёмки
- Админ может добавить категорию без изменения кода
- Сотрудник видит только активные категории
- Заявка с удалённой категорией в истории отображает label из БД

## 5. Модуль «Задачи»

**Референс Bitrix24:** задачи и проекты, роли исполнитель/наблюдатель.  
**Референс amoCRM:** задачи в карточке сделки, дедлайны.

### Функции MVP (есть)
- Список задач, статусы, приоритеты
- Назначение исполнителя

### Функции v2
- Канбан-доска (как в Bitrix24)
- Подзадачи, чек-листы
- Привязка задачи к заявке / сотруднику / проекту
- Напоминания и просрочки (как задачи amoCRM с дедлайном)
- Шаблоны повторяющихся задач

### API
- Расширить ` /api/tasks` фильтрами status, assignee, dueDate
- `POST /api/tasks/:id/comments`

## 6. Модуль «Чат»

**Референс Bitrix24:** мессенджер, чаты отделов, уведомления.  
**Референс amoCRM:** внутренние комментарии (не полноценный чат, но паттерн ленты).

### Функции MVP (есть)
- Комнаты, сообщения, история

### Функции v2
- Личные диалоги 1:1

- Вложения, упоминания @user
- Push / browser notifications
- Поиск по сообщениям

## 7. Модуль «CRM / Продажи» (новый, по образцу amoCRM)

**Зачем:** amoCRM силён в воронке продаж — SNARK может вести проекты B2B без отдельной CRM.

### Функции v1
- Сделки (лиды → переговоры → договор → закрыто)
- Карточка сделки: контакт, сумма, этап, ответственный
- Воронка с drag-and-drop (как amoCRM)
- Поля: компания, источник, теги

### Функции v2
- Интеграция почты и звонков (лог касаний)
- Автозадачи при смене этапа (Digital Pipeline)
- Дубликаты и слияние контактов

### Модель данных
- `deals`, `deal_stages`, `deal_activities`, `companies`

## 11. Сквозные требования

### Безопасность
- JWT + refresh, RBAC по ролям
- Audit log критичных действий (есть)
- Загрузка файлов: лимит размера, MIME-проверка

### UX (из Bitrix24)
- Единые статусы и бейджи во всех модулях
- Действия в один клик там, где возможно («Взять в работу», «Согласовать»)
- Пустые состояния с подсказкой следующего шага

### UX (из amoCRM)
- Карточка сущности как центр: все вкладки на одном экране
- Лента событий в хронологическом порядке
- Быстрое переключение ответственного

### Интеграции (дорожная карта)
- Email (SMTP / Exchange)
- Telegram-бот для уведомлений
- 1С:ЗУП (кадры) — выгрузка сотрудников
- Календарь (Google / Outlook)
