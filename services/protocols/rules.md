# ПРАВИЛА РАБОТЫ В ПРОЕКТЕ СНАРК-БОТ

Версия: 1.0 (09.04.2026)  
Цель: обеспечить единый стиль, безопасность и скорость разработки при работе в Cursor.

## 1. Общие принципы
- Cursor-first: 80% кода генерируется Cursor. Никогда не пиши вручную то, что можно получить через хороший промпт.
- Один PR — одна задача. Не больше 300 строк изменений.
- Всегда пиши тесты (pytest) для новых функций и эндпоинтов.
- Типизация обязательна (Python 3.12 + from __future__ import annotations).
- Безопасность ФЗ-152 — любое действие с ПДн должно проходить через существующий security модуль.
- Русский язык — все логи, комментарии, переменные, которые видит пользователь — только на русском.

text## 3. Git Workflow
- main — только рабочие релизы
- develop — основная ветка разработки
- Название ветки: feature/модуль-короткое-описание или bugfix/...
- Перед пушем: ruff check --fix, black ., mypy .
- PR должен содержать:
  - Описание задачи
  - Изменения в CHANGELOG.md
  - Скриншоты/видео работы (для модулей бота и портала)

## 4. Код-стиль
- black + ruff (конфиг уже в репозитории)
- isort
- Максимальная длина строки — 100
- Все функции и классы — с docstring (Google-style)
- Переменные: snake_case, константы: UPPER_CASE
- Классы: PascalCase

## 5. Безопасность и ФЗ-152
- Никогда не логируй ПДн (ФИО, телефон, email) в production.
- Все аудиофайлы и транскрипты совещаний — только в зашифрованном виде.
- Используй только существующий SecurityContext и PermissionDependency.

## 6. Тестирование
- Unit-тесты для всех новых функций
- Integration-тесты для 1С-коннектора и RAG
- Перед сдачей модуля — 100% покрытие новых файлов
